OpenGl学习记录(一)：创建窗口

发表于 2017-05-26

创建窗口

因为学习opengl要引入一些库，正好电脑里有cocos2dx的代码，因为cocos2dx底层是用的opengl，所以打算直接利用之。

首先创建一个cocos2dx的工程，然后找到工程的main函数入口,windows在win32/main.cpp

#include "main.h"
#include "AppDelegate.h"

USING_NS_CC;

int WINAPI _tWinMain(HINSTANCE hInstance,
                       HINSTANCE hPrevInstance,
                       LPTSTR    lpCmdLine,
                       int       nCmdShow)
{
    UNREFERENCED_PARAMETER(hPrevInstance);
    UNREFERENCED_PARAMETER(lpCmdLine);

    // create the application instance
    AppDelegate app;
    return Application::getInstance()->run();
}

改成如下即可：

#include "main.h"
#include "../Classes/AppDelegate.h"
#include "iostream"
#include "glfw3.h"


int WINAPI _tWinMain(HINSTANCE hInstance,
                       HINSTANCE hPrevInstance,
                       LPTSTR    lpCmdLine,
                       int       nCmdShow)
{
    glfwInit();
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
    GLFWwindow *windows = glfwCreateWindow(800, 600, "first windows", nullptr, nullptr);
    if (windows == NULL)
    {
        std::cout << "windows init fail" << std::endl;
        glfwTerminate();
    }
    glfwMakeContextCurrent(windows);

    glewExperimental = true;
    if (glewInit != GLEW_OK)
    {
        std::cout << "glew init fail" << std::endl;
    }
    glViewport(0, 0, 800, 600);

    while (!glfwWindowShouldClose(windows))
    {
        glfwPollEvents();
        glfwSwapBuffers(windows);
    }
    return 0;
}

总的流程就是初始化glfw，初始化glew，创建窗口，绑定窗口到opengl上下文，设置opengl视窗，在循环体内响应输入，交换缓冲区。

绑定输入回调

glfwSetKeyCallback(windows, keycallback);

回调函数的样式如下：

void keycallback(GLFWwindow *windows, int key, int code, int action, int mode)
{
    if (key == GLFW_KEY_ESCAPE && action == GLFW_PRESS)
    {
        glfwSetWindowShouldClose(windows, true);
    }
}

当操作系统的输入事件发生了，是怎样调用到此函数的，我猜想是glfwSetKeyCallback只是保留函数指针，在glfwPollEvents()的时候会去检测底层操作系统的输入事件，如果有输入则调用callback，可以发现把glfwPollEvents()函数去掉的话，回调确实没有正确调用了，以下是glfwPollEvents的注释：

  • This function processes only those events that are already in the event
  • queue and then returns immediately. Processing events will cause the window
  • and input callbacks associated with those events to be called.
    *
  • On some platforms, a window move, resize or menu operation will cause event
  • processing to block. This is due to how event processing is designed on
  • those platforms. You can use the
  • window refresh callback to redraw the contents of
  • your window when necessary during such operations.
    *
  • On some platforms, certain events are sent directly to the application
  • without going through the event queue, causing callbacks to be called
  • outside of a call to one of the event processing functions.

前半段的注释和我的猜想一致，但是最后一段

  • On some platforms, certain events are sent directly to the application
  • without going through the event queue, causing callbacks to be called
  • outside of a call to one of the event processing functions.

这段话的意思是在某些平台上，事件直接和应用进行了绑定，不需要事件处理函数。这个地方感觉很疑惑，不需要事件处理函数那就是底层会直接调用上层的函数？感觉不科学，看来还是要补习补习操作系统。
